    Introduction

    Welcome to the Embedded Systems C Gesture Library! You can use this library in your embedded systems application to interpret input from multitouch devices as gestures.

    View code snippets on the right, navigate to different sections on the left.
    This library is written in C without any operating system dependencies. All memory is allocated once at the start of program execution and never freed.

    Setup

    Environment Setup

    At the start of your code, before including "esc/gesture.h", set up the necessary environment variables

    #define ESC_ENV_VARIABLE_1 <val>
#define ESC_ENV_VARIABLE_2 <val>
...

#include "esc/gesture.h"

    Make sure to replace <val>.

    The ESC Gesture Library makes use of predefined environment variables to set up the gesture recognition system. Depending on your system, you should change these as necessary.

    Recognized Variables

    Variable Default Description
    ESCG_REMEMBER_GESTURES 1 The number of gestures to remember. This data is stored within struct ESCG::Gesture *gesture.
    ESCG_DATA_INPUT_STORAGE 45 How many data inputs from the touch device to remember. This data is stored as struct ESCG::TouchData and is used to interpret gestures.
    ESCG_MAX_CONTACT_COUNT 5 How many contacts to recognize for any given touch.
    ESCG_USE_FLOAT_COORDS undefined If defined, struct ESCG::Point will use floating point screen coordinates. Otherwise, integer coordinates will be used.
    ESCG_DEVICE_TYPE ESCG_DEVICE_CUSTOM If defined by one of ESCG_DEVICE_XXX other than ESCG_DEVICE_CUSTOM, ESCG::init() will not take a parameter.
    ESCG_USE_MULTITHREADING undefined If defined, a thread will be created to collect the data, allowing for concurrent execution without manual calls to ESCG::poll().
    Attempting to define these after including the library will result in an error, excluding ESCG_USE_MULTITHREADING. This must be defined before calling ESCG::init().

    Enabling and Using Gesture Recognition

    Before starting your execution loop, run

    ESCG::init(<adapter>)

    During your execution loop, call

    ESCG::poll([set of parameters to be passed to the data adapter])

    And to retrieve the latest gesture recognized, reference the pointer

    ESCG::gesture

    This points to an array containing the number of gestures defined by ESCG_REMEMBER_GESTURES. The head will always be the latest gesture tracked.

    There are some necessary library calls to ensure that gesture recognition works correctly. Call on ESCG::init() before running your main loop and ESCG::poll(). The single parameter to ESCG::init() sets the data adapter. If ESCG_DEVICE_TYPE has been overridden, there will be no parameter to take in. The array of arguments passed to ESCG::poll() get sent to the Adapter for the configured device. within your main loop to ensure that gesture recognition occurs. You can access recognized gestures with the pointer ESCG::gesture. This points to an array of Gestures, the count of which is determined by ESCG_REMEMBER_GESTURES. The latest recognized gesture will be first. See Data Types for more information.

    If ESCG_USE_MULTITHREADING has been defined before calling ESCG::init(), you will not need to call ESCG::poll().

    Configuring an Adapter

    We have created data adapters for certain devices. See the table at the bottom of this section for more information.

    If you intend to use a device we haven't included an adapter for, you will need to create your own. An adapter should be a function that takes a single argument of type *void and returns a TouchData. The argument array is passed from ESCG::poll(). This function should establish contact with the touch device and convert its data.

    Premade adapters

    Adapters have been made for the following devices (set ESCG_DEVICE_TYPE accordingly):

    B Device ESCG_DEVICE_B
    C Device ESCG_DEVICE_C

    Data Types

    The ESC Gesture Library utilizes several data types that are exposed to a user. Such datatypes are defined in

    namespace ESCG.

    Gesture

    Contains information about recognized gestures. There are automatically created and populated into ESCG::gesture when valid gestures are recognized. Gestures are only pushed back after contact has been released and reset. That is, if a swipe is being executed, the data at ESCG::gesture[0] will change, but nothing else will.

    Fields:

    • type (int) – a number representing the type of gesture found. You can assert this value against predefined values ESCG_GESTURE_XXX.
    • startData (ESCG::TouchData) – the ESCG::TouchData recognized at the start of the gesture.
    • endData (ESCG::TouchData) – the ESCG::TouchData recognized at the end of the gesture.

    TouchData

    A container for holding contact data.

    Fields:

    • numContacts (int) – how many contacts are contained here.
    • contacts (ESCG::Point*) – an array of points of contact.
    • timestamp (long)

    Point

    A container holding an X/Y coordinate.

    Fields:

    • x (int)
    • y (int)
    If ESCG_USE_FLOAT_COORDS has been defined, x and y will be of type float.
